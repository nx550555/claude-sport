# 自己チェックプロトコル（監視機能）

> **Claude Codeへの指示：**
> - タスク完了後・応答作成前に必ずこのチェックリストを通過させる
> - 1項目でも「×」があれば、応答前に修正または明示的に報告する
> - チェック結果を応答に含める必要はないが、問題があれば必ず言及する

---

## CHECK-1: 基本方針チェック（方針からの脱線防止）

以下を確認する：

- [ ] **オッズ取得はユーザー提供JSONを起点にしているか？**（Claudeが自分でオッズを探していないか）
- [ ] **統計データ（xGF%・NRtg・PD等）はClaudeが自分で取得したか？**（ユーザーに求めていないか）
- [ ] **スクリーニングは全試合対象か？**（live/completed/upcomingすべて）
- [ ] **記録はJSONファイルに書き込んだか？**（チャット内だけで終わっていないか）
- [ ] **ルール更新が必要な場合、rules_{sport}.jsonに書き込んだか？**
- [ ] **累積統計の更新が必要な場合、stats/cumulative.jsonを更新したか？**
- [ ] **ダッシュボード更新が必要な場合、dashboard/index.htmlを更新したか？**
- [ ] **時刻はJSTで解釈しているか？**（ユーザー提供データは原則JST）

---

## CHECK-2: 情報精度チェック（ハルシネーション防止）

以下を確認する：

- [ ] **統計値（xGF%・NRtg・PD等）は実際に取得したデータに基づいているか？**（記憶・推測・概算で補っていないか）
- [ ] **チーム名・オッズの対応は正確か？**（team1/team2の順番と odds の対応が逆になっていないか）
- [ ] **試合日・ラウンド番号は正確か？**（ファイル内の日付誤記を鵜呑みにしていないか）
- [ ] **記録ファイルの参照は実際に読み込んで確認したか？**（記憶だけで「〜のはず」と言っていないか）
- [ ] **EV計算式は正しいか？**（EV = 推定勝率 × オッズ - 1）
- [ ] **信頼度75%・EV+5%の両方を満たしているかを確認したか？**
