## 4. モバイル高確率予想テーブル表示修正

### 問題の経緯
- 英語: 横スクロールで正常表示
- 日本語: 文字が縦に並ぶ（PENDING → P/E/N/D/I/N/G）、横スクロールなし

### 根本原因
HTML テーブルは日本語文字（どこでも折り返し可能）を利用してセルを縮め、ビューポート内に収めようとする。英語はアルファベット単語が途中で折れないため自然に横スクロールになる。

### 試行過程
1. `table-layout:fixed` + `word-break:break-all` + 列幅指定 → PENDINGが縦になる（前セッションで導入）
2. カードレイアウト（各行を縦並びカードに変換）→ 見やすいがユーザーは横スクロール希望
3. 専用CSS全削除 → 改善なし（原因は別）

### 最終解決策
```css
/* モバイル全般 */
.badge { white-space: nowrap; }  /* バッジテキストが縦にならない */

/* モバイル高確率予想テーブル専用 */
@media(max-width:600px) {
  #content-highprob .data-table { min-width: 680px; }
  /* テーブルを強制的にビューポートより広くする → 横スクロール発動 */
}
```

- 日本語・英語両モードで横スクロール統一
- 他テーブル（全履歴等）は影響なし
