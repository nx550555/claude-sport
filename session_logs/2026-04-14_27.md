## 実施内容

### 1. CLAUDE_CODE_START.md 【絶対禁止】架空情報セクション修正

**変更前（誤り）:**
```
選手名・チーム名・スコア・オッズ・怪我・出場情報など、いかなる具体的な情報も架空で作ることは絶対禁止。
「例」「イメージ」「サンプル」という理由でも許可されない。
```

**変更後（正しい意図を反映）:**
```
選手名・チーム名・スコア・オッズ・怪我・出場情報など、架空の具体的情報を生成することは絶対禁止。

情報の明示ルール（必ず守ること）：
- 実際に検索・確認した情報 → そのまま具体的に記載する
- サンプル・フォーマット例を示す場合 → 必ず「【サンプル】」と明記し、プレースホルダーを使う
- 確認していない情報 → 「未確認」と書く
- 不明な情報 → 「不明」と書く

絶対にやってはいけないこと：
- 実在しない選手名・怪我・出場情報・スコア・オッズを具体的な値として生成する
- 「例として」「イメージとして」という理由で架空の具体的情報を提示する（プレースホルダーを使わず）
- 未確認の情報を確認済みであるかのように記載する
```

**修正理由:** 「サンプルも禁止」という記述はユーザーの意図と乖離していた。ユーザーの実際の意図は「サンプルならサンプルと明記すること」であり、【サンプル】ラベル＋プレースホルダーを使えばサンプル表示は可。

### 2. memory/feedback_no_fabrication.md 修正

同内容をmemoryファイルにも反映。
- サンプルは【サンプル】ラベル＋プレースホルダーで可（禁止ではない）に訂正
- 未確認→「未確認」、不明→「不明」の明示ラベリングルールを追記
